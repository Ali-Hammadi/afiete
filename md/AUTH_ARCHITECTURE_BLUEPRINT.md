# Afiete Authentication (Auth) Feature - Complete Architecture Blueprint

**Status**: Architecture Planning Document  
**Scope**: Multi-step signup, login, password recovery, profile completion, session management, account deletion  
**Stack**: Flutter (Clean Architecture + Cubit) ↔ Django Backend (Token-based authentication)  

---

## 1. Architecture Overview

### 1.1 Core Principles
- **Token-Based Authentication**: Access + Refresh token pair managed via `TokenStorage`
- **Progressive Signup**: Username → Email → Password → OTP Verification → Profile Completion
- **Flexible Profile Data**: DOB, Gender, Phone can be `null` initially; required to complete signup but optional for returning users
- **Forced Completion**: If user logs in with incomplete profile data, app redirects to profile completion screen
- **Session Consistency**: Tokens cached locally; logout clears all cached data; app state reset to "fresh install" state
- **Error Normalization**: Server errors mapped to professional user-facing messages

### 1.2 Data Flow Diagram
```
┌──────────────┐
│  Signup Flow │
└──────┬───────┘
       │
       ├─→ Step 1: Enter (nickname, email, password)
       │   │
       │   └─→ Backend: Validate email uniqueness → Generate OTP → Send email
       │   │
       │   └─→ App: Cache `_pendingSignupUser`, show OTP verification screen
       │
       ├─→ Step 2: Verify OTP
       │   │
       │   └─→ Backend: Validate OTP code → Set is_verified=true
       │   │
       │   └─→ **CRITICAL**: Call login endpoint to receive access_token
       │   │              (OTP verification does NOT return token)
       │   │
       │   └─→ App: Cache token via TokenStorage → Emit AuthLoaded state
       │
       └─→ Step 3: Complete Profile (DOB, Gender, Phone)
           │
           └─→ Backend: PATCH profile with authenticated request
           │
           └─→ App: Navigate to Home (auth verification complete)

┌─────────────────────┐
│  Login/Forgot PWD   │
└──────────┬──────────┘
           │
           ├─→ Email + Password → Backend validates
           │   │
           │   └─→ Success: Return access_token + profile
           │   │
           │   └─→ Check if profile is complete (DOB/Gender/Phone filled)
           │   │
           │   └─→ If incomplete: Redirect to profile completion screen
           │   │
           │   └─→ If complete: Navigate to Home
           │
           └─→ Forgot Password:
               ├─→ Email → OTP email
               ├─→ Verify OTP + New Password
               └─→ Auto-login with new credentials

┌────────────────────────┐
│  Session Management    │
└──────────┬─────────────┘
           │
           ├─→ Logout: Clear tokens + Clear app state → Splash screen
           │
           └─→ Delete Account: Verify (email + password) → Hard delete → Clear all data
```

```dart
class UserAuthEntity extends Equatable {

  final String? nickname;          // User-set alias
  final String email;

  final String? profileImageUrl;   // Optional avatar
  final String? accessToken;       // JWT token
  final String? refreshToken;      // Refresh token

  const UserAuthEntity({
    required this.id,
    required this.username,
    this.nickname,
    required this.email,
    this.dateOfBirth,
    this.gender,
    this.phoneNumber,
    required this.isVerified,
    this.profileImageUrl,
    this.accessToken,
    this.refreshToken,
  });

  @override
  List<Object?> get props => [
    id, username, nickname, email, dateOfBirth, gender,
    phoneNumber, isVerified, profileImageUrl, accessToken, refreshToken,
  ];

  /// Check if profile is complete (required for post-signup onboarding)
  bool get isProfileComplete =>
      dateOfBirth != null && gender != null && phoneNumber != null;
```

```dart
class AuthRemoteDataSourceImpl implements AuthRemoteDataSource {
  static const String _baseUrl = '/api/auth/';
  final Dio _dio;

  AuthRemoteDataSourceImpl(this._dio);

  @override
  Future<OtpModel> signup({
    required String nickname,
    required String email,
    required String password,
  }) async {
    try {
      final response = await _dio.post(
        '${_baseUrl}signup/',
        data: {
          'nickname': nickname,
          'email': email,
          'password': password,
        },
      );
      if (response.statusCode == 201 || response.statusCode == 200) {
        return OtpModel.fromJson(response.data);
      }
      throw DioException(requestOptions: response.requestOptions);
    } on DioException {
      rethrow;
    } catch (e) {
      throw DioException(requestOptions: RequestOptions(path: '${_baseUrl}signup/'));
    }
  }

  @override
  Future<UserModel> verifySignupOtp({
    required String email,
    required String otpCode,
```

```dart
      return Left(ServerFailure.fromDioError(e));
    } catch (e) {
      return Left(ServerFailure(e.toString()));
    }
  }

  @override
  Future<Either<Failure, OtpEntity>> requestEmailChangeOtp() async {
    try {
      final remoteOtp = await remoteDataSource.requestEmailChangeOtp();
      return Right(remoteOtp.toEntity());
    } on DioException catch (e) {
      return Left(ServerFailure.fromDioError(e));
    } catch (e) {
      return Left(ServerFailure(e.toString()));
    }
  }

  @override
  Future<Either<Failure, void>> confirmEmailChange({
    required String newEmail,
    required String otpCode,
  }) async {
    try {
      await remoteDataSource.confirmEmailChange(
        newEmail: newEmail,
        otpCode: otpCode,
      );
      return const Right(null);
    } on DioException catch (e) {
      return Left(ServerFailure.fromDioError(e));
    } catch (e) {
      return Left(ServerFailure(e.toString()));
```

### 5.1 Error Mapping (core/error/failure.dart)

```dart
abstract class Failure extends Equatable {
  final String errorMessage;
  const Failure(this.errorMessage);

  @override
  List<Object?> get props => [errorMessage];
}

class ServerFailure extends Failure {
  ServerFailure(super.errorMessage);

  factory ServerFailure.fromDioError(DioException dioError) {
    switch (dioError.type) {
      case DioExceptionType.connectionTimeout:
      case DioExceptionType.receiveTimeout:
        return ServerFailure('Connection timed out. Check your internet.');

      case DioExceptionType.badResponse:
        return ServerFailure._fromResponse(
          dioError.response?.statusCode,
          dioError.response?.data,
        );

      case DioExceptionType.unknown:
        if (dioError.message?.contains('SocketException') ?? false) {
          return ServerFailure('No internet connection');
        }
        return ServerFailure('An unexpected error occurred');

      default:
        return ServerFailure('Network error occurred');
    }
  }

  factory ServerFailure._fromResponse(int? statusCode, dynamic response) {
    final errorData = response is Map ? response['error'] : null;
    final message = errorData?['message'] ?? response?['message'] ?? '';

    switch (statusCode) {
      case 400:
        // Bad request - validation error
        if (message.toLowerCase().contains('email')) {
          return ServerFailure('This email is already registered');
        }
        if (message.toLowerCase().contains('password')) {
          return ServerFailure('Password does not meet requirements');
        }
        return ServerFailure(message.isNotEmpty ? message : 'Invalid request');

      case 401:
        // Unauthorized - invalid credentials
        if (message.toLowerCase().contains('inactive') ||
            message.toLowerCase().contains('deactivated') ||
            message.toLowerCase().contains('disabled') ||
            message.toLowerCase().contains('suspended')) {
          return ServerFailure(
            'Your account is restricted and cannot sign in',
          );
        }
        return ServerFailure(message.isNotEmpty ? message : 'Authentication failed');

      case 403:
        // Forbidden - account restricted
        return ServerFailure('Access denied. Please contact support.');

      case 404:
        // Not found
        return ServerFailure('The requested resource was not found');

      case 429:
        // Rate limited
        return ServerFailure(
          'Too many attempts. Please try again in a few minutes.',
        );

      case 500:
      case 502:
      case 503:
        return ServerFailure('Server error. Please try again later.');

      default:
        return ServerFailure(
          message.isNotEmpty ? message : 'An error occurred. Please try again.',
        );
    }
  }
}

class CacheFailure extends Failure {
  CacheFailure(super.errorMessage);
}

class ValidationFailure extends Failure {
  ValidationFailure(super.errorMessage);
}
```

- After `verifySignupOtp()` returns user with access_token, **immediately cache it** before calling `updateProfileInfo()`
- The Dio interceptor must attach the cached token to all authenticated requests
- Implement refresh token logic to auto-renew expired access tokens

### Profile Completion
- Treat `dateOfBirth`, `gender`, `phoneNumber` as optional for profile entity
- But in signup flow, they are **required** before navigation to home
- If returning user has incomplete profile, force redirect on login

### OTP Flow Reusability
- Same OTP screen used for: signup verification, forgot password, email change
- Pass `flow` parameter to determine which cubit method to call on verification

### Error Normalization
- Never show raw Django error messages (e.g., "user matching query does not exist")
- Always map to professional, user-friendly text
- Special handling for account restriction errors (inactive/suspended)

## 9. Django Backend Contracts (Reference)

### Endpoint Specifications
```
POST /api/auth/signup/
Body: { nickname, email, password }
Response: { email, expires_in }

POST /api/auth/verify-signup-otp/
Body: { email, otp_code }
Response: { id, username, email, access_token, refresh_token, ... }

POST /api/auth/login/
Body: { email, password }
Response: { id, username, email, access_token, refresh_token, ... }

GET /api/auth/profile/
Auth: Bearer {access_token}
Response: { id, username, email, date_of_birth, gender, phone_number, ... }

PATCH /api/auth/profile/
Auth: Bearer {access_token}
Body: { date_of_birth?, gender?, phone_number? }
Response: { id, username, email, date_of_birth, gender, phone_number, ... }

POST /api/auth/logout/
Auth: Bearer {access_token}
Response: {}

DELETE /api/auth/delete-account/
Auth: Bearer {access_token}
Body: { email, password }
Response: {}

POST /api/auth/forgot-password/
Body: { email }
Response: { email, expires_in }

POST /api/auth/verify-forgot-password-otp/
Body: { email, otp_code, new_password }
Response: { email, expires_in } or auto-login response?
```

## Notes for Development

1. **Use part files for state definitions**: `auth_state.dart` as part of `auth_cubit.dart`
2. **Keep cubit methods focused**: One async method per business action
3. **Never emit states directly**: Always use `result.fold()` pattern for Either handling
4. **Cache tokens immediately**: After login/signup OTP verification
5. **Handle incomplete profile**: Check `user.isProfileComplete` after login
6. **Normalize errors**: Use `ServerFailure._fromResponse()` for all HTTP errors
